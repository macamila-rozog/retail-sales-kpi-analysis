# Retail Sales KPI Analysis & Reporting

## Project Overview
This project presents an end-to-end retail sales analysis focused on **KPI definition, validation, and reporting**.  
The objective is to evaluate overall sales performance, identify trends, and produce a structured report suitable for operational and executive decision-making.

The project follows a realistic analytics workflow: data exploration and KPI validation in Python, reproducible metric logic in SQL, structured reporting in Excel, and high-level visualization in Tableau.


## Business Context
Retail organizations rely on consistent and reliable performance indicators to understand revenue behavior, customer purchasing patterns, and product performance.

## Key KPIs Analyzed

### A. Global Sales KPIs
- **Total Revenue** – Sum of all transaction amounts  
- **Total Transactions** – Count of completed transactions  
- **Average Order Value (AOV)** – Revenue divided by number of transactions  
- **Average Items per Order** – Mean quantity per transaction  
- **Minimum and Maximum Order Value** – Outlier and risk identification  

### B. Time-Based KPIs
- **Transactions per Day**  
- **Revenue per Day**  
- **Average Order Value over Time**

### C. Product Performance KPIs
- **Revenue by Product Category**  
- **Units Sold by Category**  
- **Average Price per Order by Category**

### D. Customer Demographics KPIs
- **Revenue by Gender**  
- **Average Order Value by Gender**  
- **Average Order Value by Age Group**

### E. Transaction Distribution KPIs
- **Distribution of Low, Medium, and High-Value Orders**  
- **Top Transactions by Value**
